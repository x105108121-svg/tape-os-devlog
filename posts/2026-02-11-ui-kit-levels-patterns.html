UI kit, /ui, /patterns и уровни Next.js

Мы допилили ship так, чтобы любой новый деплой выглядел “взросло” по умолчанию. Цель — не “красивости”, а скорость проверки идей без потери качества.

  • UI kit: токены, типографика, карточки, кнопки, формы.
  • /ui: галерея компонентов (чтобы собирать интерфейс как из лего).
  • Темы: clean / premium / playful (переключатель хранит выбор).
  • /patterns: готовые секции лендинга (hero, logos, features, proof, pricing, compare, FAQ, CTA, footer).
  • Levels 1–4: от “лендинг+интерактив” до “product demo skeleton”.

Быстрый тестовый деплой (Level 4): открыть демо (смотри /ui и /patterns прямо там).

Команды (одна и та же механика, разный уровень):

  • ship/run.sh app my-demo — level 1 (по умолчанию)
  • ship/run.sh app my-demo 2 — API + форма
  • ship/run.sh app my-demo 3 — Supabase (опционально)
  • ship/run.sh app my-demo 4 — onboarding + state + event log + export

Важно: скорость не должна делать демо “сырой заготовкой”. Теперь базовые штуки (типографика, интервалы, фокус-стейты) уже на месте.

Дальше будем наращивать “маркетинг-слой” (ещё 2–3 hero‑варианта, таблицы, proof‑секции) и параллельно улучшать продуктовые паттерны для level 2–4.
